# SICA: A Self-Improving Coding Agent

**Citation:** Robeyns, Szummer, Aitchison — ICLR 2025 Workshop on Scaling
Self-Improving Foundation Models; arXiv:2504.15228. Code:
MaximeRobeyns/self_improving_coding_agent.

## Mechanism

Eliminates the distinction between meta-agent and target agent: **one agent
edits its own codebase** to improve its cost, speed, and benchmark
performance. The shipped base agent is deliberately minimal (no fancy edit
tools, no LSP/tree-sitter) but has the building blocks to bootstrap those
features and specialize itself to the benchmark task distribution. The repo
includes an `oversight/` module and event/callgraph instrumentation — the
safety and observability scaffolding is part of the design, not an
afterthought.

## What agent-kit takes from it

- The **minimal-base + bootstrap** stance mirrors agent-kit's "calm defaults"
  vision: ship the smallest correct harness, grow it from evidence.
- An `agent_change_log.md` at the root of the self-modifying codebase is a
  cheap, durable lineage artifact — agent-kit's equivalent is the lore commit
  history, but a per-surface changelog is worth considering for the
  harness-surface manifest.
