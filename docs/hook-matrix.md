# Hook Matrix

This matrix separates what the upstream tool documents from what agent-kit
currently scaffolds and verifies in this repo.

## Routing ownership

- Agent-kit owns `ak_*` dev-workflow routing.
- context-mode owns `ctx_*` routing when that plugin is installed.
- `.omx/` is runtime/state only. It is not a direct hook surface.

## Claude Code

**Documented upstream**

- Claude hooks live in `.claude/settings.json`.
- Supported events include `SessionStart`, `PreToolUse`, `PostToolUse`,
  `UserPromptSubmit`, and `Stop`.
- `Edit`, `Write`, and `MultiEdit` are distinct Claude tool matchers.

**Agent-kit scaffolds and verifies**

- `ak-sessionstart-routing`
- `ak-pretool-guard`
- `ak-post-tool`
- `ak-guard-switch`
- `ak-stop-qa`
- `worktree.symlinkDirectories` includes `.claude`
- Claude `PreToolUse` and `PostToolUse` matchers include `MultiEdit`

## Codex

**Documented upstream**

- Codex supports project hook configuration via `.codex/hooks.json`.
- We do not currently rely on upstream docs for richer per-tool matcher claims
  beyond the hook file contract itself.

**Agent-kit scaffolds and verifies**

- `.codex/hooks.json` is generated by `ak setup`
- events: `SessionStart`, `PreToolUse`, `PostToolUse`, `UserPromptSubmit`, `Stop`
- edit matcher contract used by agent-kit today: `Edit|Write`

**Explicit non-claim**

- agent-kit does **not** currently claim Codex `MultiEdit` parity
- agent-kit does **not** currently claim richer post-edit automation beyond the
  generated `Edit|Write` hook surface

## context-mode

**Ownership rule**

- If context-mode is installed, it owns `ctx_*` routing nudges.
- Agent-kit does not duplicate those `ctx_*` instructions in its own routing
  block.
