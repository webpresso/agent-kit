## Follow-up policy

- Prefer updating a durable how-to/runbook only when user-facing behavior changed.
- Keep bot bumps, version-package PRs, format-only cleanups, and test-only flake fixes in this audit note unless they alter an operator command or merge gate.
- When a PR fixes a CodeQL or Code Quality finding, add a local `wp audit security-quality-regressions` rule only for high-signal smells that can be detected without reimplementing the analyzer.
