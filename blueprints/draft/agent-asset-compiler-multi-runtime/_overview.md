# Agent-asset compiler — multi-runtime

## Product wedge anchor

- **Stage outcome:** Cite VISION.md's north star ("One command, fully wired" — `ak setup` turns a bare git checkout into a repo where every AI coding agent has the context, hooks, and guardrails it needs) and the Elegance Pass 2026 stage outcome ("every webpresso public package + ingest-lens consumer reaches fully wired agent surfaces from a fresh clone via one command, with CI catching drift"). This blueprint replaces the current Mac-only symlinker with a portable compiler so the north star actually holds on Linux/Windows and across six AI-agent runtimes instead of two — and gives consumers a curator (`ak skills orphans`) so the global skill listing stays clean over time.
- **Consuming surface:** CLI verbs `ak sync` (rewritten), new `ak skills {list,prune,orphans,import,budget,migrate-from-symlinks}` family, and new audits `ak audit gitignore-agent-surfaces` and `ak audit memory-unified`.
- **New user-visible capability:** A developer can author `.agent/skills/foo/SKILL.md` once and have it appear correctly (and only once) in Claude's skill listing, Codex's `$skill` surface, OpenCode's subagent picker, Cursor's MDC index, Windsurf's rules layer, and Gemini's commands — *and* run `ak skills orphans --fix` to delete files in any of those six generated dirs that lack a `~/.agent/` source (the q-* class of pollution). Plus: project AGENTS.md sections can be **structurally merged** across user-global, project-root, and per-directory layers with deterministic output and a `merged.provenance.json` showing which file contributed each section.

## Why this exists

Today's reality (verified 2026-05-11 against ingest-lens + monorepo on disk):

- `.claude/skills` is a **symlink** to `../.agent/skills` — Mac-only, breaks on Windows, confuses `git status`, and quietly inverts on accidental edits to either side.
- `.claude/commands/` (19 files in ingest-lens, 24 in monorepo) and `.claude/agents/`, `.codex/agents/` (41 TOML files in ingest-lens) are **hand-curated per tool** — no canonical source, no drift detection.
- `.opencode/` doesn't exist in either consumer.
- `~/.claude/commands/` had 12 unsourced `q-*` files from the `agentic-stack` repo polluting the global skill listing across every project, eating the 1%-of-context budget and triggering the truncation warning that started this work.
- Each tool has a different filesystem layout, frontmatter schema, enable/disable mechanism, and skill-listing budget — fact-checked against current docs (see Appendix A).

A single canonical source compiled by `ak sync` lets every consumer (monorepo, ingest-lens, future public extractions) ship one set of assets and get correct, drift-free outputs in every tool — and lets pruning happen in one place.

## Non-goals

- Not a generic "agent framework" — strictly an asset compiler + curator.
- Not redesigning the `.agent/` canonical schema beyond adding `agents/` (new) and `memory.merge.yaml` (new, optional) — existing `commands/`, `skills/`, `rules/`, `workflows/`, `guides/` stay where they are.
- Not supporting backwards compatibility with the symlink-based sync. **Hard cutover at agent-kit v0.11.0.** Existing `ak cursor-windsurf-sync` command is **deleted** in the same commit (its logic is folded into the new compiler).
- Not generating prose inside any `AGENTS.md` — the compiler merges sections from existing files and writes the merged result; it never invents content.

## Architecture

### Canonical source (committed)

```
.agent/                              ← canonical, committed
├── skills/<name>/SKILL.md           ← model-invokable workflows
├── commands/<name>.md               ← user-invokable slash commands
├── agents/<name>.md                 ← specialized subagents (NEW dir)
├── memory/                          ← NEW: AGENTS.md layered sources
│   ├── AGENTS.md                    ← project-root base layer (optional override of repo-root AGENTS.md)
│   ├── memory.merge.yaml            ← optional: section directives + frontmatter merge (RFC 7396)
│   └── <subdir>/AGENTS.md           ← optional: per-directory overlays (matches Codex's walk semantics)
├── rules/, workflows/, guides/      ← unchanged (existing reference content)
└── hooks/<event>.json               ← optional, per-project hooks
AGENTS.md                            ← committed; the **merger output** lives here (Codex+OpenCode native primary)
CLAUDE.md                            ← committed; user-owned, expected to contain `@AGENTS.md`
```

### Generated outputs (gitignored, regenerated by `ak sync`)

```
# Claude Code
.claude/skills/, /commands/, /agents/    ← from .agent/{skills,commands,agents}/ (copy, not symlink)
.claude/settings.local.json              ← unchanged (Claude's own gitignore convention)
.claude/settings.json                    ← team-committed (NOT generated)

# Codex CLI
.agents/skills/                          ← Codex skill path (verified: `.agents`, not `.codex`)
.codex/agents/*.toml                     ← from .agent/agents/*.md via md→TOML transform
.codex/hooks.json                        ← from .agent/hooks/*.json via wrapped-form transform
.codex/config.toml                       ← team-committed (NOT generated)

# OpenCode
.opencode/agents/<name>.md               ← from .agent/{skills,agents}/ (mode mapping)
.opencode/commands/<name>.md             ← from .agent/commands/*.md (passthrough)
.opencode/opencode.json                  ← team-committed (NOT generated)

# Cursor
.cursor/rules/<name>.mdc                 ← from .agent/skills/<name>/SKILL.md (MDC frontmatter transform)
.cursor/commands/<name>.md               ← from .agent/commands/*.md (passthrough)

# Windsurf
.windsurf/rules/<name>.md                ← from .agent/skills/<name>/SKILL.md (passthrough)
.windsurfrules                           ← deprecated; only written if present in legacy consumer (warning emitted)

# Gemini CLI (folded from existing `ak cursor-windsurf-sync`)
.gemini/commands/<name>.toml             ← from .agent/commands/*.md via md→TOML transform

# Merger artifacts
AGENTS.md                                ← merger output (committed; downstream-tool primary)
.agent/.merged.provenance.json           ← gitignored; maps each output section → contributing file
.agent/.sync-manifest.json               ← gitignored; content-hash sentinel for drift detection
```

User-global scope mirrors the same shape at `~/.agent/` → `~/.claude/`, `~/.agents/skills/`, `~/.codex/agents/`, `~/.config/opencode/`, `~/.cursor/`, `~/.codeium/windsurf/`, `~/.gemini/`.

### Source → target mapping

| Source | Claude | Codex | OpenCode | Cursor | Windsurf | Gemini |
|---|---|---|---|---|---|---|
| `.agent/skills/foo/SKILL.md` | `.claude/skills/foo/SKILL.md` | `.agents/skills/foo/SKILL.md` | `.opencode/agents/foo.md` (`mode: subagent`) | `.cursor/rules/foo.mdc` | `.windsurf/rules/foo.md` | *skip* (Gemini has no skill concept) |
| `.agent/commands/foo.md` | `.claude/commands/foo.md` | *skip* (no FS-loaded commands) | `.opencode/commands/foo.md` | `.cursor/commands/foo.md` | *skip* (no commands surface) | `.gemini/commands/foo.toml` (md→TOML) |
| `.agent/agents/foo.md` | `.claude/agents/foo.md` | `.codex/agents/foo.toml` (md→TOML) | `.opencode/agents/foo.md` (`mode: primary`) | *skip* (no agents surface) | *skip* | *skip* |
| `.agent/memory/**/AGENTS.md` + `memory.merge.yaml` | merged → `AGENTS.md` at root (then user `@AGENTS.md` imports into CLAUDE.md) | merged → `AGENTS.md` (native) | merged → `AGENTS.md` (native) | n/a (uses MDC rules) | n/a (uses rules) | merged → `AGENTS.md` (Gemini reads root AGENTS.md per docs) |

Codex skip for custom commands and Cursor/Windsurf/Gemini skips for agents/skills/commands are intentional — verified against each tool's current docs (no documented surface as of 2026-05). The upgrade path for cross-tool reach is to rename `commands/foo.md` → `skills/foo/SKILL.md`, which then compiles everywhere that has a skill concept.

### AGENTS.md merging (Option C hybrid, per 2026-05-11 research)

Background: every shipped agent runtime today either (a) naively concatenates layered AGENTS.md files root→leaf (Codex), (b) does file-level first-match-wins (OpenCode), (c) uses `@import` directives (Claude), or (d) ignores layering entirely. **No existing tool does section-keyed structural merging.** Sources for these claims: developers.openai.com/codex/guides/agents-md, opencode.ai/docs/rules, code.claude.com/docs/en/memory.

The compiler introduces section-keyed merging with three layers of escalation:

**Layer 1 — Default behavior (no config required):** Parse each `.agent/memory/**/AGENTS.md` to mdast. Key top-level `##` sections by slugified heading. Deeper layer's section replaces parent's section atomically (the whole `##` block). Missing sections inherit. Frontmatter merged via RFC 7396 semantics (recursive object merge; `null` deletes a key). This is Option A from the research — covers 90% of cases with zero new syntax.

**Layer 2 — Explicit directives via `memory.merge.yaml`:** Sibling YAML to `AGENTS.md` lets a layer override section semantics:

```yaml
# .agent/memory/memory.merge.yaml
sections:
  - heading: Build
    op: replace      # default; explicit for clarity
  - heading: Conventions
    op: append       # appended to parent; useful for additive rules
  - heading: Legacy
    op: delete       # explicitly null out an inherited section
frontmatter:        # RFC 7396 merge patch
  model: opus
  experimental: null  # deletes inherited key
```

Directives are out-of-band so the bare `AGENTS.md` files stay valid for any naive concatenator (Codex sees prose; ignores YAML siblings).

**Layer 3 — Observable provenance:** Every `ak sync` writes `.agent/.merged.provenance.json` mapping each output section's slug → contributing source file path + op. `ak skills list --memory` surfaces this. Reproducible: same inputs → byte-identical output.

**Precedence order** (root → leaf, deeper wins per section):

1. `~/.agent/memory/AGENTS.md` (user-global base)
2. `~/.codex/AGENTS.md` (Codex's user-global, imported as user-overlay)
3. Project-root `.agent/memory/AGENTS.md`
4. Per-directory `.agent/memory/<subdir>/AGENTS.md` (walked from CWD up; closer wins)
5. `~/.agent/memory/AGENTS.override.md` (user-global override)
6. Project-root `.agent/memory/AGENTS.override.md` (project override)

Step 2 and the `.override.md` files match Codex's existing override chain (developers.openai.com/codex/guides/agents-md) so a Codex user's existing layering migrates without surprises.

### CLAUDE.md handling

`ak sync` does **not** generate `CLAUDE.md`. It runs `ak audit memory-unified` which warns if the merged `AGENTS.md` exists and `CLAUDE.md` does not `@AGENTS.md`-import it. User owns CLAUDE.md content — typical Claude-specific prose plus the one-line `@AGENTS.md` import.

## Technology Choices

| Decision | Choice | Reasoning |
|---|---|---|
| Sync mechanism | **Copy** with content-hash sentinel (`.agent/.sync-manifest.json`) | Symlinks broke on Windows + confused git; copy is portable, drift-detectable, single direction (.agent → tools) |
| md → TOML transform for Codex subagents + Gemini commands | Shared TOML emitter at `src/compiler/_toml.ts` (extracted from existing `src/symlinker/unified-sync.ts` before deletion) | One transformer, two consumers; deterministic key order; round-trip-safe for required fields (F1) |
| Memory merging | **Option C hybrid** — section-keyed override as default + optional `memory.merge.yaml` directives + provenance JSON | Research (2026-05-11) found no existing tool does structural agent-context merging; Codex/OpenCode/Claude all concatenate naively. Option A semantics keep dumb runtimes compatible; Option C upgrades reproducibility + observability without polluting the markdown (F2, F5) |
| Codex skill listing budget | Lint description length (cap at 800 chars per skill) in `ak audit skill-budget` | Codex hardcodes ~2% ctx / 8000 chars total; can't raise via settings (F3) |
| Backwards compat | **None** — major bump to v0.11.0 | Symlink → copy transition can't run side-by-side; migration command (`ak skills migrate-from-symlinks`) is a one-shot. `ak cursor-windsurf-sync` is deleted in the same commit. |
| User-global scope | `~/.agent/` → `~/.claude/`, `~/.agents/skills/`, `~/.codex/agents/`, `~/.config/opencode/`, `~/.cursor/`, `~/.codeium/windsurf/`, `~/.gemini/` | Mirrors project-level mental model; avoids special-casing global commands (which is how the q-* pollution accumulated) |
| OpenCode "skill" mapping | Compile to subagent (`mode: subagent`); **first-class adapter, default-on** | OpenCode has no skill concept; subagent is the closest invocable analog. Promoting from flagged to default because dogfooding inside agent-kit ships with the same release (F4, F6) |
| Cursor MDC transform | YAML-frontmatter → MDC-frontmatter shape per cursor.com/docs/rules (globs, alwaysApply, description). SKILL.md `description` becomes MDC `description`; `when_to_use` becomes `globs` if globs-shaped, else inlined into body | Cursor's MDC is YAML-compatible but uses different key names; documented transform is lossless for the subset agent-kit emits (F7) |
| Windsurf rules transform | Passthrough md to `.windsurf/rules/<name>.md` per windsurf.com/docs/cascade/rules. **No legacy `.windsurfrules` write path — single-file format is deleted by `ak skills migrate-legacy` during migration.** | Per "zero backwards compat for any integration" — keeping the legacy surface alive perpetuates two-file ambiguity. Migration deletes `.windsurfrules` after merging its content into a `.windsurf/rules/_migrated.md` rule once, then never again. (F8) |
| Legacy integration shims | **None** — `ak skills migrate-legacy` is a one-shot that deletes every legacy artifact: `.claude/skills` symlinks, `.windsurfrules`, `.cursorrules`, single-file `.codex/AGENTS.md` if duplicated by `.agent/memory/`, stale Gemini sync outputs, the old `ak cursor-windsurf-sync` invocation in any consumer package.json scripts | "0 backwards compat for any integration" applies recursively — any keep-the-legacy-path-alive code is a future eviction problem and a documentation-debt source. Detect once, delete once, never look back. |
| Gemini md → TOML | Fold existing `ak cursor-windsurf-sync` Gemini path into the new compiler at `src/compiler/adapters/gemini.ts`. Delete old code. | One compiler, one transform set; matches "no backwards compat" rule |
| Section-merger AST library | `remark` + `remark-frontmatter` + `mdast-util-frontmatter` (already in tree if any markdown work touches them; otherwise add as deps) | Well-maintained, mdast is the de-facto standard; no published "merge two mdast trees by heading" plugin exists so we write the merge layer ourselves (F2 confirmation) |
| Frontmatter merge semantics | RFC 7396 (JSON Merge Patch) | Recursive object merge, `null` deletes a key, widely understood, smallest spec surface |

## Edge Cases

| ID | Severity | Case | Mitigation |
|---|---|---|---|
| E1 | HIGH | User hand-edits `.claude/skills/foo/SKILL.md` instead of `.agent/skills/foo/SKILL.md` — edit is lost on next sync | Manifest tracks content hash per generated file; `ak sync` aborts with a diff if any generated file has drifted, prompts `ak skills import .claude/skills/foo` or `--force` |
| E2 | HIGH | Skill `foo` exists both in `.agent/skills/foo/` AND `~/.agent/skills/foo/` (project shadows user) | Project wins (matches Claude's enterprise > personal > project but for our scope: project > user). Document precedence in README. |
| E3 | MEDIUM | Reverse migration: existing `.codex/agents/*.toml` files in consumers need to import into `.agent/agents/*.md` (md is source-of-truth) | `ak skills import-all` runs a TOML → md transform; lossy fields emit warnings, not silent drops |
| E4 | MEDIUM | Codex `.agents/skills/` path collides with hypothetical `.agents/` dir used for other purposes | Verified against current Codex docs; if a future Codex release changes the path, version-pin the path in adapter config |
| E5 | MEDIUM | Hook file `.codex/hooks.json` is currently committed in consumers and includes hand-edits | One-shot import: `ak skills import` reads existing `.codex/hooks.json`, converts back to `.agent/hooks/*.json`, then `ak sync` regenerates. Document the round-trip is lossless. |
| E6 | LOW | OpenCode subagent compiled from a SKILL.md may have frontmatter mismatch (SKILL.md uses `allowed-tools`, OpenCode uses `tools: { write: false }`) | Adapter maps `allowed-tools` (denylist style) → OpenCode `tools` (per-tool boolean) via well-defined transform; unmappable fields warn |
| E7 | LOW | `ak sync` runs in parallel from two terminals on same repo (rare) | Manifest write uses `O_EXCL` lock file; second invocation sees lock and exits with clear message |
| E8 | CRITICAL | First v0.11.0 `ak sync` in a consumer with the current symlinked `.claude/skills` deletes the user's `.agent/skills/` because it walks the symlink in reverse | Migration command `ak skills migrate-from-symlinks` is required as preflight; `ak sync` refuses to run if symlinks are detected and no manifest exists |
| E9 | HIGH | Two memory layers both define `## Build` — section-keyed merge replaces atomically with deeper layer's content; user expected concat | Default is `op: replace` (deeper wins) per Option A semantics; users wanting append must add `memory.merge.yaml` with `op: append`. Document in README with a worked example. |
| E10 | MEDIUM | `memory.merge.yaml` references a section heading that doesn't exist in any layer | Merger emits warning naming the unused directive; doesn't fail (allows directives to anticipate future sections) |
| E11 | MEDIUM | Merged `AGENTS.md` output is committed but provenance JSON is gitignored — reviewer can't see which file contributed each section | `ak skills list --memory --json` reads the provenance file; `ak audit memory-provenance` runs in CI to ensure provenance matches current output (fails if out of sync) |
| E12 | LOW | MDC frontmatter for Cursor expects `globs` as array; SKILL.md's `when_to_use` is free-form prose | Adapter parses `when_to_use` only if it's a YAML-shaped glob array (e.g. `["**/*.ts"]`); otherwise inlines into MDC body and emits info-level message |
| E13 | LOW | Cursor renames `.cursor/rules/*.mdc` to `.cursor/rules/*.md` in a future release | Adapter is version-aware: writes whichever extension the consumer's installed Cursor version expects (capture via `cursor --version` if available; otherwise default to `.mdc` per current docs) |
| E14 | MEDIUM | Windsurf's legacy single-file `.windsurfrules` exists at v0.11.0 migration time | `ak skills migrate-legacy` reads existing `.windsurfrules`, ingests its content into `.windsurf/rules/_migrated.md` (one-time), then **deletes** `.windsurfrules`. `ak sync` thereafter never writes the single-file surface. Same pattern for `.cursorrules` if present. |
| E15 | MEDIUM | Consumer's `package.json` has `"ak cursor-windsurf-sync"` in scripts or husky hooks | `ak skills migrate-legacy` scans `package.json` scripts, `.husky/*`, and CI yamls for the removed command. Rewrites them to `ak sync` or removes them if redundant. Logged in migration report. |
| E16 | LOW | Old `.codex/AGENTS.md` (Codex's user-global pattern) duplicates content the user has also placed in `.agent/memory/AGENTS.md` | `ak skills migrate-legacy` diffs the two; if identical, keeps the canonical `.agent/memory/` copy and deletes `.codex/AGENTS.md`. If different, the merger now imports both as separate layers per precedence chain — but the user is warned to consolidate. |

## Risks

| ID | Severity | Risk | Mitigation |
|---|---|---|---|
| R1 | HIGH | Codex's `.agents/skills/` path is a documented but unusual root-level directory; some tools (rg ignore, vite watcher) may need to be told about it | Add `.agents/` to ignore patterns in `ak setup --with base-kit`'s common-ignore template; document in README |
| R2 | HIGH | Section-keyed merging is a novel pattern — no shipped agent runtime does it. Risk that consumers expect concatenation and are surprised by atomic-replace defaults. | Default Option A (no merge.yaml needed) → backward-compatible with reading the merged output. Explicit "merging cheatsheet" in README with worked examples. `ak audit memory-provenance` makes the merge visible in PR review. |
| R3 | MEDIUM | TOML round-trip (md → TOML → md) for Codex subagent / Gemini command reverse-import (E3) loses comments and key ordering | Document in migration log; user must review imported files. Comments and ordering are best-effort, not guaranteed. |
| R4 | MEDIUM | "No backwards compat" means consumers must coordinate the bump — monorepo and ingest-lens both pin agent-kit GitHub SHAs/catalog versions | Land in agent-kit first; create one PR per consumer that runs migration + bumps pin in the same commit |
| R5 | LOW | `ak audit gitignore-agent-surfaces` may false-positive on consumers with unusual gitignore layouts | Use POSIX glob match, not regex; allow `#ak:ok` line marker to silence |
| R6 | MEDIUM | Six runtime adapters means six independent breakage surfaces if upstream paths change (e.g. Cursor moves `.cursor/rules/` to `.cursor/instructions/`) | Each adapter pins the verified path constant in a single file (`src/compiler/adapters/_paths.ts`); doc fetcher (`ak audit runtime-paths`) periodically refetches official docs and diffs against constants, surfacing drift as a CI warning |
| R7 | MEDIUM | `remark`/`mdast` adds dependency weight to agent-kit (was lean) | Bench startup time before/after; if regression > 50ms, switch to a lighter custom mdast subset (we only need heading-section parsing, not full GFM) |
| R8 | LOW | Cursor/Windsurf adoption may be lower than expected inside webpresso — fold-in scope is partly speculative | Adapter tests use real fixtures, not synthetic ones; if Cursor/Windsurf usage drops, the adapter still ships but moves to `optional-targets` table |

## Migration plan (no backwards compat)

Sequence:

1. **Land agent-kit v0.11.0** with new compiler + `ak skills migrate-from-symlinks`. Old symlink sync code AND the standalone `ak cursor-windsurf-sync` command are **removed** in the same commit (no dual paths).
2. **Cut release** to GitHub Packages (`@webpresso/agent-kit@0.11.0`) and capture commit SHA for GitHub-pinned consumers.
3. **Per consumer (monorepo, ingest-lens):**
   1. Bump agent-kit pin (catalog version for ingest-lens; workspace dep for monorepo).
   2. Run `ak skills import-all` (one-shot; sweeps existing `.claude/commands/`, `.claude/agents/`, `.codex/agents/`, `.cursor/rules/`, `.windsurf/rules/`, `.gemini/commands/`, plus any AGENTS.md layering at user-global and project-root, into `.agent/`).
   3. Run `ak skills migrate-legacy` (one-shot; hard-deletes every legacy artifact: symlinks, `.windsurfrules`, `.cursorrules`, duplicate `.codex/AGENTS.md`, stale Gemini outputs, removed-command-surface references in package.json/husky/CI yamls).
   4. Run `ak sync` (regenerates all six runtime targets + merged AGENTS.md + provenance JSON from canonical source).
   5. Update `.gitignore` via `ak setup --with base-kit` (re-run is idempotent; expands existing block to cover cursor/windsurf/gemini paths).
   6. Verify with `ak audit gitignore-agent-surfaces` + `ak audit memory-unified` + `ak audit memory-provenance`.
   7. Commit: one Lore-Commit-Protocol commit per consumer documenting the cutover.

Both consumer migrations land in the same week to keep workspace coherent.
